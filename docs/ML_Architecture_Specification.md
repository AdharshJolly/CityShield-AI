# CityShield AI: Unified ML Architecture Specification

This document serves as the Principal ML Architectural blueprint. It strictly enforces reproducibility, dataset versioning, experiment tracking, and module decoupling across the entire machine learning lifecycle, from raw image ingestion to model deployment.

---

The `ml_engine` is divided into strict domains: datasets (preparation) and training (execution).

```text
CityShield/
├── data/
```

## 2. Dataset Pipeline Architecture

### Execution Flow
`harmonize.py` → `validate.py` → `split_dataset.py` → `generate_yaml.py` → `statistics.py`

### Dataset Versioning Strategy
Every run of the pipeline produces an immutable, versioned folder (e.g., `data/processed/cityshield_v1/`). 
- Future iterations (e.g., adding scraped trees) will produce `cityshield_v2`.
- `generate_yaml.py`, `statistics.py`, and `trainer.py` dynamically ingest the target version via CLI args (e.g., `--version v1`).

### Modules & Specifications

#### A. `taxonomy.py`
**Purpose:** The absolute Single Source of Truth for class mappings. No duplicated strings anywhere in the repository.
**Dependencies:** None.
```python
# taxonomy.py
CITYSHIELD_CLASSES = [
    "fire", "smoke", "burning_waste", "fallen_tree", "collapsed_structure",
    "streetlight_normal", "streetlight_damaged", "animal", "vehicle", "person"
]
```

#### B. `harmonize.py`
**Purpose:** Re-maps chaotic class IDs into the `taxonomy.py` format and executes dataset capping logic.
**Dependencies:** Imports from `taxonomy.py`. Output goes to `data/interim/`.

#### C. `validate.py`
**Purpose:** Drops corrupted OpenCV images, deletes out-of-bound bounding boxes (`<0.0` or `>1.0`), and removes orphaned labels. Output remains in `data/interim/`.

#### D. `split_dataset.py`
**Purpose:** Distributes the interim data into stratified `train/val/test` folders inside the versioned processed directory, preserving class distributions.
- **Inputs:** `data/interim/`
- **Outputs:** `data/processed/cityshield_v1/` (`train/`, `val/`, `test/`)
- **Rules:** Uses fixed random seeds (e.g., `42`) to guarantee identical reproducible splits. Default ratio is 70% Train, 20% Val, 10% Test.

#### E. `generate_yaml.py`
**Purpose:** Programmatically builds `data.yaml` inside the versioned folder.
- **Dependencies:** Imports `CITYSHIELD_CLASSES` from `taxonomy.py`.

#### F. `statistics.py`
**Purpose:** Calculates final bounding box metrics and generates the formal Dataset Manifest.

## 4. Training Architecture Specification (`ml_engine/training/`)

This module separates hyperparameter definitions from the training loop, ensuring reproducibility.

- **`config.py`**: Defines dataclasses/Pydantic models for YOLO hyperparameters (epochs, batch size, learning rate, imgsz).
- **`experiment_tracker.py`**: Wrapper for Weights & Biases (W&B) and local JSON metric logging. Manages the `experiments/expXYZ/` folder creation.
- **`callbacks.py`**: Hooks injected into the YOLO engine (e.g., custom Early Stopping based on mAP50-95, or dynamic LR scheduling).
- **`trainer.py`**: The main execution loop. Instantiates `config`, loads the target `dataset_version`, attaches `callbacks`, and initiates `model.train()`.
- **`evaluator.py`**: Evaluates the `best.pt` model against the `test/` split explicitly. Generates PR curves and confusion matrices.
- **`inference_benchmark.py`**: Benchmarks `best.pt` on a dummy video to prove the model executes under the strict `<150ms` latency constraint.

**Execution Order:** `trainer.py` -> `evaluator.py` -> `inference_benchmark.py`.

## 5. Experiment Tracking Specification

Every execution of `trainer.py` spawns a new deterministic folder under `/experiments/`. 

```text
experiments/
└── exp001/
    ├── weights/          # best.pt, last.pt
    ├── logs/             # events.out.tfevents (Tensorboard)
    ├── metrics.json      # Final JSON dump of mAP, Precision, Recall, F1
    ├── config.yaml       # Snapshot of the hyperparameters used
    └── benchmark.json    # Inference latency metrics (e.g., 85ms/frame)
```

**Tracked Metrics (Mandatory):**
- Dataset Version, Model Size (e.g., YOLOv12-L).
- Hyperparameters.
- `mAP50`, `mAP50-95`, `Precision`, `Recall`, `F1`.
- Training Duration (Hours), GPU VRAM utilization.
- Final Inference Latency (ms/frame).

## 6. Training Readiness Gate Checklist

To prevent wasted GPU hours and protect the architecture, **Model Training may only begin when the following criteria are met:**

- [ ] **A. Dataset Pipeline Complete:** Data exists physically in `data/processed/cityshield_v1/`.
- [ ] **B. Split Integrity:** `train`, `val`, and `test` directories exist with a 70/20/10 ratio.
- [ ] **C. Single Source of Truth:** `taxonomy.py` is locked and imported correctly.
- [ ] **D. YAML Generation:** `cityshield_v1/data.yaml` matches the frozen taxonomy exactly.
- [ ] **E. Manifest Generated:** `dataset_manifest.json` is fully populated with class counts and split metrics.
- [ ] **F. Training Code Approved:** `trainer.py` and `experiment_tracker.py` are written, reviewed, and merged to `dev`.

**Conclusion:** Once this checklist passes, the Lead Architect is authorized to execute `python ml_engine/training/trainer.py --version v1 --exp exp001`.
